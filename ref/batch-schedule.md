# 批量智能排期 — 详细算法

> 功能三 (batch-schedule) 的完整执行步骤。
> 配置变量含义见 SKILL.md，值来自 `ref/config.yaml`。

## 输入

| 参数 | 说明 | 示例 |
|------|------|------|
| `range` | 排期时间范围 | `4/20 - 4/24`、`下周` |
| `tasks` | 二选一：① 直接指定需求名称/ID ② 指定迭代名称（自动筛选） | `["需求A","需求B"]` 或 `"版本一"` |

## Step 1 — 获取任务列表

- 直接指定需求：使用用户提供的列表
- 指定迭代名称：MQL 查询该迭代下、当前节点进行中的任务：

```sql
SELECT `name`, `work_item_id`
FROM `MY_PROJECT_NAME`.`story`
WHERE array_contains(in_progress_nodes_name(), 'MY_NODE_NAME')
  AND `planning_sprint` = '迭代名称'
```

展示结果给用户确认，数量多于预期时请用户裁剪。

## Step 2 — 计算工作日

```
W = 时间范围内工作日数（排除周六日和法定节假日）
```

节假日处理：若排期范围包含法定节假日（如五一、国庆、春节等），扣除对应天数。
若 `ref/holidays.yaml` 存在则读取；否则提示用户确认范围内是否有节假日。

示例：4/28(Mon) ~ 5/9(Fri)，含五一假期 5/1~5/5，调休 4/27(Sun)上班：
  日历工作日 = 10，扣除 5/1~5/5 中的 3 个工作日，加回 4/27 调休 = 10 - 3 = 7（4/27 不在范围内则不加）

## Step 3 — 分配估分

```
N      = 任务数量
TARGET = W + 0.5              // 目标总分，略高于工作日数
base   = ceil_half(TARGET / N) // 每个任务的基础估分，最小 0.5
```

**`ceil_half` 定义**：向上取整到最近的 0.5

```
ceil_half(x) = ceil(x × 2) / 2
示例：ceil_half(1.1) = 1.5,  ceil_half(2.0) = 2.0,  ceil_half(0.3) = 0.5
```

调整逻辑：

```
若 sum(points) > W + 1：
  从估分最大的任务开始缩减 0.5，直到 sum ≤ W + 1

若仍有零头：最后一个任务补齐（可用 0.2 / 0.3）
```

优先使用 0.5 和 1，凑数时才用 0.2/0.3。

## Step 4 — 分配排期（交错排列）

核心：span 比估分多 1.5 工作日，让相邻任务真正交叉（重峦叠嶂），而非首尾相接。

```
slot_width = W / N

for 任务 i（从 0 开始）:
  span         = points[i] + 1.5
  start_offset = i * slot_width
  end_offset   = start_offset + span - 1

  start_date = range_start 后第 floor(start_offset) 个工作日
  end_date   = range_start 后第 min(floor(end_offset), W-1) 个工作日
```

效果示意（W=5，N=4，各 1pt，slot_width=1.25，span=2.5）：

```
      Mon  Tue  Wed  Thu  Fri
任务1: [=========]
任务2:      [=========]
任务3:           [=========]
任务4:                [=========]
```

每个任务与相邻任务约重叠 1.25 天（50%）。

为什么 +1.5 而非 +1：span=2 时重叠仅 0.75 天（几乎只是触碰）；span=2.5 时重叠达 1.25 天，视觉效果自然。

## Step 5 — 展示方案，等待用户确认

```
以下排期方案即将写入，请确认：

| 需求   | 排期          | 估分 | 区间天数 |
|--------|---------------|------|---------|
| 任务A  | 4/20 - 4/21   | 1    | 2       |
| 任务B  | 4/21 - 4/22   | 1    | 2       |
| 任务C  | 4/22 - 4/23   | 1    | 2       |
| 任务D  | 4/23 - 4/24   | 1.5  | 2       |

总估分：4.5（目标 5~6）
确认执行？
```

用户确认后再执行写入。

## Step 6 — 执行更新

对每个任务调用 `update_node`（见 SKILL.md 更新节点规范）。
所有日期转换为 ms 时间戳，参考 `ref/timestamps.md`。

> 若某项 `update_node` 失败，记录错误原因，继续处理其余任务。

## Step 7 — 验证

调用 `list_schedule` 验证，确认总分在 [W, W+1] 区间。

```
|--------|---------------|------|---------|------|

| 任务B  | 4/21 - 4/22   | 1    | 2       | ✅   |
| 任务C  | 4/22 - 4/23   | 1    | 2       | ✅   |
| 任务D  | 4/23 - 4/24   | 1.5  | 2       | ❌ reason |
```
