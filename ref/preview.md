# 排期预览 — 详细渲染规则

> 功能四 (preview) 的完整执行步骤。
> 配置变量含义见 SKILL.md，值来自 `ref/config.yaml`。

## Step 1 — 获取两周排期数据

并行调用两次 `list_schedule`：
- 本周：当前自然周 Mon~Sun
- 下周：下一自然周 Mon~Sun
- 参数：`project_key=MY_PROJECT_KEY`，`user_keys=[MY_USER_KEY]`，`work_item_type_keys=["_all"]`

## Step 2 — 获取迭代信息

对所有 work_item_id 批量查询：

```sql
SELECT `name`, `work_item_id`, `planning_sprint`
FROM `MY_PROJECT_NAME`.`story`
WHERE `work_item_id` IN (id1, id2, ...)
```

> 注意：迭代字段 key 是 `planning_sprint`，不是 `迭代`。

按 `planning_sprint` 分组；无迭代的归入「未关联迭代」组放在最后。

## Step 3 — 临期预警

飞书项目需求无独立"截止日期"字段，**用节点排期结束日期代替**：

| 条件 | 标记 |
|------|------|
| 排期结束日在本周内 | `⚠本周截止` |
| 排期结束日在下周内 | `📌下周截止` |
| 排期结束日已过今日 | `🔴已逾期` |

## Step 4 — 渲染文本甘特图

整个甘特图包裹在 ` ``` ` 代码块中，确保等宽字体渲染。

### 名称列对齐算法

汉字 = 2 显示宽，ASCII/符号 = 1 显示宽，目标显示宽 = **22**：

```
display_width(s) = Σ( 2 if '\u4e00'≤c≤'\u9fff' else 1  for c in s )

if display_width(name) > 20:
    截断直到 display_width ≤ 20，末尾加 "…"
else:
    name_col = name + " " * (22 - display_width(name))
```

### 每行格式

```
  {name_col}{W1_bar} | {W2_bar}  {pts}{warn}
```

- 每格固定 `███`/`░░░`（3字符），日期间 1 空格
- 本周 / 下周用 ` | ` 分隔
- 临期标记紧跟估分后
